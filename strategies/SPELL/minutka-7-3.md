# Минутка 7.3

## Название

SPELLUSDT 1m SHORT regime-filtered max income

## Идея

Минутка 7.3 - это агрессивная версия ветки 7.x.

Новый лимит риска: максимальная просадка до `20%`.

Цель: увеличить доходность относительно 7.2, сохранив базовые ограничения:

- доходность выше нуля на 7, 30, 90, 180 и 365 днях;
- win rate выше `85%`;
- максимальная просадка не выше `20%`.

## Основные параметры

| Параметр | Значение |
|---|---:|
| Пара | `SPELLUSDT` |
| Таймфрейм | `1m` |
| Направление | `SHORT` |
| Вход | Лимитный у текущей цены |
| Размер позиции | `24%` текущего баланса |
| Комиссия | `0.02%` вход + `0.02%` выход |
| Проскальзывание | `0` |
| Фильтр сигнала | `short_score >= 40` |
| Тейк | `0.28%` |
| Стоп | `4.0%` |
| Выход по времени | `120 минут` |
| Дневной автостоп | `2%` |

## Фильтр рыночного режима

Сделка разрешена только если на закрытой сигнальной свече выполнены все условия:

| Фильтр | Значение |
|---|---:|
| ATR% | `>= 0.25%` |
| Дистанция close от EMA200 | от `-1.5%` до `+1.5%` |
| Доходность за 7 дней | от `-40%` до `+10%` |

## Команда

```bash
python3 scripts/gala_mb_backtest.py --market futures_archive --days 365 --warmup-days 7 --direction short \
  --entry-mode maker_limit \
  --limit-entry-offset-pct 0 \
  --limit-entry-timeout-min 1 \
  --fee-pct 0.0002 \
  --slippage-pct 0 \
  --position-pct 0.24 \
  --short-threshold 40 \
  --short-tp-pct 0.0028 \
  --short-sl-pct 0.040 \
  --short-time-stop-min 120 \
  --daily-loss-stop-pct 0.02 \
  --filter-atr-min-pct 0.0025 \
  --filter-dist-ema200-min -0.015 \
  --filter-dist-ema200-max 0.015 \
  --filter-return-7d-min -0.40 \
  --filter-return-7d-max 0.10
```

## Вывод

Минутка 7.3 сильно увеличивает доходность относительно 7.2: годовой результат
растет с `+30.56%` до `+52.70%`.

Но это почти предельная версия по риску. Максимальная просадка на годе `19.99%`,
то есть минутка буквально упирается в новый лимит `20%`.

Практический вывод: 7.3 можно считать максимальной доходной исследовательской
версией при идеальном maker-исполнении. Для реальной торговли она слишком
чувствительна: даже небольшое ухудшение комиссии или slippage сразу выводит ее
за лимит просадки.
