# MOVR / CHZ / MANA Monthly 4% Portfolio

Дата фиксации: 2026-05-08

Цель этой версии - не максимальная годовая доходность, а режим cashflow: каждый полный месяц забирать минимум `$40` с рабочего капитала `$1000`.

Это портфельная версия вокруг `MOVR 1h LONG Monthly Protected`. MOVR сам по себе дает аккуратные, но редкие сигналы, поэтому к нему добавлены CHZ и MANA, чтобы закрывать пустые месяцы.

## Логика

| Параметр | Значение |
|---|---:|
| База капитала | `$1000` |
| Monthly target | `$40` / `4%` |
| Monthly loss stop | `2%` |
| MOVR вес | `50%` |
| CHZ вес | `25%` |
| MANA вес | `25%` |
| Scale / риск-множитель | `2.5x` |
| Вход | maker-limit с откатом `0.05%` |
| Комиссия | maker `0.02%` за сторону |
| Slippage | `0` в базовом тесте |
| Плечо | фактически нужен повышенный exposure; это не чистый spot-вариант |

## Используемые модули

| Монета | Модуль |
|---|---|
| MOVR | `MOVR 1h LONG Monthly Protected` |
| CHZ | `CHZ LONG Best` |
| MANA | `MANA LONG Best` |

## Альтернативы

Более доходный, но менее спокойный вариант: `CHZ 20% / MOVR 80%`, scale `2.5x`, monthly loss stop `3%`.

Он дал `11/11` месяцев с `$40+`, net `$589.95`, MaxDD `8.20%`, PF `2.67`. Минус - сильнее завязан на две монеты и требует большего допуска по месячной просадке.

## Вывод

Это первая найденная MOVR-центричная версия, которая реально дотянула до цели `4% в месяц` на проверенном окне.

Но проверка на более длинном окне показала, что это не стабильная 3-летняя cashflow-стратегия.

## Проверка на 36 месяцев

Проверка: `2023-05` - `2026-04`, те же веса `MOVR 50% / CHZ 25% / MANA 25%`, scale `2.5x`, monthly target `$40`, monthly loss stop `2%`.

| Метрика | Значение |
|---|---:|
| Месяцев с `$40+` | `3/36` |
| Плюсовых месяцев | `21/36` |
| Минусовых месяцев | `15` |
| Net result | `$177.38` |
| MaxDD | `29.60%` |
| Profit Factor | `1.12` |
| Худший месяц | `-$77.14` |
| Лучший месяц | `$63.44` |
| Сделок | `728` |

Вывод после 36 месяцев: стратегия не подходит как постоянная цель `4% каждый месяц`. Она хорошо попала в свежий участок `2025-06` - `2026-04`, но до этого была длинная плохая фаза, особенно в `2024`.

Теперь статус стратегии: **recent-window candidate / не core cashflow**.

Дальше нужен контроль:

- stress комиссии и исполнения;
- свежая проверка `24h / 7d / 30d`;
- проверка на более длинной истории, если по модулям хватает данных;
- paper-режим перед реальными деньгами.
