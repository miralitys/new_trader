# Operational Monitor

Это рабочая схема, что делать дальше с найденными стратегиями. Смысл не в том, чтобы все включить сразу, а в том, чтобы каждый день получать простое решение: **торговать / наблюдать / выключить**.

## Слои

| Слой | Что внутри | Что делаем |
|---|---|---|
| Лучшие стратегии | GALA 7.3, GALA 10, GALA 11.2, ONE 11.2, CHZ, SHIB, JASMY, SAND, MANA, ANKR, SPELL | Проверяем свежие 30/60 дней и stress исполнения. Включаем только те, где здоровье не сломалось. |
| Cashflow | GALA20/SPELL80 4%, ONE20/SPELL80 10% high-risk | Отдельный месячный режим. Не смешивать с compounded-стратегиями. |
| Regime monitor | RIF core, ACH/REZ watchlist | Торговать только если прошлые 30/60 дней проходят health-check. |
| Wave monitor | GUA, IRYS, B2, MEGA, RAVE | Не торговать постоянно. Ждать hot-сигнал и включать на короткое окно. |

## Главный порядок

1. Сначала каждый день обновляется статус всех монет.
2. Потом отбираются только разрешенные стратегии.
3. Если одна и та же монета есть в нескольких стратегиях, выбирается один режим, а не все сразу.
4. Если стратегия не проходит health-check, она не удаляется, а переводится в watchlist.
5. Если стратегия ломается в stress исполнения, она не идет в live.

|---|---|---|
| RIF | [RIF Live Readiness](rif-live-readiness-2026-05-05.md) | Главный подтвержденный кандидат: проходит health, stress и paper execution. |

| Блок | Монеты / стратегии | Почему не core |
|---|---|---|
| ONE | Минутка 11.2 | Рабочая, но отдельно проверять свежий режим и не смешивать с high-risk портфелем. |
| CHZ | Минутка 10 CHZ Best | Нужен свежий stress исполнения перед live. |
| SHIB | SHIB LONG Best | Запас к комиссии небольшой. |
| JASMY | JASMY SHORT Best | Есть доходность, но win rate ниже, чем хотелось. |
| SAND / MANA | LONG Best | Низкая активность, 30 дней может давать мало сделок. |
| ACH / REZ | Regime watchlist | Похожи на RIF, но намного слабее. |
| GUA / IRYS / B2 | Wave monitor | Это не постоянная торговля, а ловля волн. |

## Ежедневное Решение

| Статус | Условие | Действие |
|---|---|---|
| TRADE | 30/60d здоровье нормальное, PF выше порога, DD в лимите, есть сделки | Можно торговать в paper/live-small. |
| WATCH | Часть условий нормальная, но мало сделок или слабый PF | Только наблюдать, без реального входа. |
| OFF | 30/60d минус, PF ниже 1, DD разрослась, stress плохой | Не торговать до восстановления. |

## Правило Против Перегруза

Если одновременно активны несколько стратегий на одной монете:

- для GALA не включаем live, пока paper execution не подтверждает плюс;
- для SPELL выбираем либо одиночную SPELL SHORT, либо cashflow-портфель, но не оба риска одновременно;
- для wave-монет не держим постоянную торговлю, только окно после hot-сигнала;
- ACH/REZ не торгуем как core, пока они не покажут свежую устойчивость.

## Что Нужно Автоматизировать

Нужен один ежедневный запуск, который сохраняет таблицу:

| Поле | Смысл |
|---|---|
| symbol | Монета |
| monitor_group | best_strategy / cashflow / regime_monitor / wave_monitor |
| strategy | Название стратегии |
| status | TRADE / WATCH / OFF |
| reason | Почему такой статус |
| return_30d / return_60d | Свежая доходность |
| pf_30d / pf_60d | Свежий profit factor |
| dd_30d / dd_60d | Свежая просадка |
| trades_30d / trades_60d | Достаточно ли сделок |

Реестр для такого монитора уже сохранен в `data/operational_monitor_universe_2026-05-04.csv`.

Первая версия с учетом stress исполнения сохранена здесь: [Operational Daily Monitor v2](operational-daily-monitor-v2-2026-05-04.md).

С 2026-05-05 monitor учитывает еще и paper execution override: если стратегия зеленая по monitor, но минусовая в paper-журнале, она снижается до `WATCH`.

## Вывод

Дальше проект должен перейти из режима “ищем красивую доходность” в режим **операционного контроля**.

Самый разумный порядок:

1. Основной paper-monitor: RIF.
2. Watchlist-monitor: GALA, SPELL, ONE, CHZ, SHIB, JASMY, SAND, MANA, ANKR, ACH, REZ.
3. Отдельный wave-monitor: GUA, IRYS, B2.
4. Cashflow отдельно: GALA20/SPELL80 4%.

Так мы не теряем найденные стратегии, но и не включаем все подряд без контроля.
