# Минутка 7.1

## Название

GALAUSDT 1m SHORT regime-filtered income

## Идея

Минутка 7.1 - это более доходная версия минутки 7.

Главное отличие:

- размер позиции увеличен с `10%` до `15%`;
- фильтр режима упрощен;
- цель остается прежней: быть в плюсе на 7, 30, 90, 180 и 365 днях, держать
  win rate выше `85%` и максимальную просадку ниже `13%`.

## Основные параметры

| Параметр | Значение |
|---|---:|
| Пара | `GALAUSDT` |
| Таймфрейм | `1m` |
| Направление | `SHORT` |
| Вход | Лимитный у текущей цены |
| Размер позиции | `15%` текущего баланса |
| Комиссия | `0.02%` вход + `0.02%` выход |
| Проскальзывание | `0` |
| Фильтр сигнала | `short_score >= 40` |
| Тейк | `0.25%` |
| Стоп | `3%` |
| Выход по времени | `180 минут` |
| Дневной автостоп | `2%` |

## Фильтр рыночного режима

Сделка разрешена только если на закрытой сигнальной свече выполнены все условия:

| Фильтр | Значение |
|---|---:|
| ATR% | `>= 0.25%` |
| Дистанция close от EMA200 | от `-1.5%` до `+1.5%` |
| Доходность за 7 дней | от `-40%` до `+10%` |

Смысл фильтра: брать SHORT только тогда, когда цена находится близко к EMA200,
волатильность достаточная, но нет экстремального недельного движения.

## Команда

```bash
python3 scripts/gala_mb_backtest.py --market futures_archive --days 365 --warmup-days 7 --direction short \
  --entry-mode maker_limit \
  --limit-entry-offset-pct 0 \
  --limit-entry-timeout-min 1 \
  --fee-pct 0.0002 \
  --slippage-pct 0 \
  --position-pct 0.15 \
  --short-threshold 40 \
  --short-tp-pct 0.0025 \
  --short-sl-pct 0.030 \
  --short-time-stop-min 180 \
  --daily-loss-stop-pct 0.02 \
  --filter-atr-min-pct 0.0025 \
  --filter-dist-ema200-min -0.015 \
  --filter-dist-ema200-max 0.015 \
  --filter-return-7d-min -0.40 \
  --filter-return-7d-max 0.10
```

## Вывод

Минутка 7.1 проходит заданные условия на всех проверочных окнах:

- доходность везде положительная;
- win rate везде выше `85%`;
- максимальная просадка на 365 днях `11.24%`, то есть ниже лимита `13%`.

По доходности минутка 7.1 заметно лучше минутки 7: годовой результат вырос
с `+2.26%` до `+25.32%`.

Главный риск: запас по просадке уже небольшой. До лимита `13%` остается около
`1.76%`, поэтому повышать размер позиции выше `15%` без стресс-теста нельзя.
Следующий обязательный шаг - проверка ухудшенной комиссии, проскальзывания и
неполного maker-исполнения.

## Стресс-тест исполнения

Стресс-тест показал, что минутка 7.1 чувствительна к комиссии и качеству
исполнения. Базовая версия работает только при очень дешевом maker-исполнении.

## Финальный вывод после стресс-теста

Минутка 7.1 остается интересной, но только при условии реального maker-fee
около `0.02%` и почти нулевого slippage. Если комиссия становится `0.03%` за
сторону или появляется заметное проскальзывание, преимущество почти исчезает, а
просадка выходит за лимит `13%`.

Практический вывод: 7.1 нельзя запускать в taker-режиме. Перед реальной
торговлей нужно проверить фактический maker-fill, реальные комиссии аккаунта и
долю неисполненных лимитных заявок.
