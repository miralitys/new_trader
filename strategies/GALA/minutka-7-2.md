# Минутка 7.2

## Название

GALAUSDT 1m SHORT regime-filtered higher income

## Идея

Минутка 7.2 - это более доходная версия минутки 7.1.

Главное отличие от 7.1:

- тейк увеличен с `0.25%` до `0.28%`;
- стоп увеличен с `3.0%` до `3.5%`;
- выход по времени сокращен со `180 минут` до `120 минут`;
- размер позиции остается `15%`.

Цель: увеличить доходность, сохранив базовые ограничения:

- доходность выше нуля на 7, 30, 90, 180 и 365 днях;
- win rate выше `85%`;
- максимальная просадка ниже `13%`.

## Основные параметры

| Параметр | Значение |
|---|---:|
| Пара | `GALAUSDT` |
| Таймфрейм | `1m` |
| Направление | `SHORT` |
| Вход | Лимитный у текущей цены |
| Размер позиции | `15%` текущего баланса |
| Комиссия | `0.02%` вход + `0.02%` выход |
| Проскальзывание | `0` |
| Фильтр сигнала | `short_score >= 40` |
| Тейк | `0.28%` |
| Стоп | `3.5%` |
| Выход по времени | `120 минут` |
| Дневной автостоп | `2%` |

## Фильтр рыночного режима

Сделка разрешена только если на закрытой сигнальной свече выполнены все условия:

| Фильтр | Значение |
|---|---:|
| ATR% | `>= 0.25%` |
| Дистанция close от EMA200 | от `-1.5%` до `+1.5%` |
| Доходность за 7 дней | от `-40%` до `+10%` |

## Команда

```bash
python3 scripts/gala_mb_backtest.py --market futures_archive --days 365 --warmup-days 7 --direction short \
  --entry-mode maker_limit \
  --limit-entry-offset-pct 0 \
  --limit-entry-timeout-min 1 \
  --fee-pct 0.0002 \
  --slippage-pct 0 \
  --position-pct 0.15 \
  --short-threshold 40 \
  --short-tp-pct 0.0028 \
  --short-sl-pct 0.035 \
  --short-time-stop-min 120 \
  --daily-loss-stop-pct 0.02 \
  --filter-atr-min-pct 0.0025 \
  --filter-dist-ema200-min -0.015 \
  --filter-dist-ema200-max 0.015 \
  --filter-return-7d-min -0.40 \
  --filter-return-7d-max 0.10
```

## Вывод

Минутка 7.2 дает больше доходности, чем 7.1, и формально проходит условия:
плюс на всех окнах, win rate выше `85%`, просадка ниже `13%`.

Но это более агрессивная версия. Запас до лимита просадки всего около `0.71%`,
а стресс-тест показывает, что даже небольшое ухудшение комиссии или slippage
выводит минутку за лимит.

Практический вывод: 7.2 можно держать как доходную исследовательскую версию, но
для реальной торговли она требует почти идеального maker-исполнения.
