# Пятиминутка 1

## Название

GALAUSDT 5m SHORT maker scalp

## Идея

Пятиминутка 1 - это прямой перенос Минутки 1 с таймфрейма `1m` на `5m`.

Торговая логика не оптимизировалась:

- тот же SHORT-сигнал;
- тот же лимитный maker-вход;
- тот же тейк;
- тот же стоп;
- тот же выход по времени в минутах.

Важно: `time-stop 90 минут` на `5m` означает примерно 18 пятиминутных свечей,
а не 90 свечей.

## Параметры

| Параметр | Значение |
|---|---:|
| Пара | `GALAUSDT` |
| Таймфрейм | `5m` |
| Направление | `SHORT` |
| Вход | Лимитный у текущей цены |
| Размер позиции | `100%` текущего баланса |
| Комиссия | `0.02%` вход + `0.02%` выход |
| Проскальзывание | `0` |
| Тейк | `0.25%` |
| Стоп | `2%` |
| Выход по времени | `90 минут` |
| Фильтр сигнала | `short_score >= 55` |

## Команда

```bash
python3 scripts/gala_mb_backtest.py --market futures_archive --symbol GALAUSDT --interval 5m --days 365 --direction short \
  --entry-mode maker_limit \
  --limit-entry-offset-pct 0 \
  --limit-entry-timeout-min 5 \
  --fee-pct 0.0002 \
  --slippage-pct 0 \
  --position-pct 1.0 \
  --short-threshold 55 \
  --short-tp-pct 0.0025 \
  --short-sl-pct 0.020 \
  --time-stop-min 90
```

## Вывод

Пятиминутка 1 не является рабочей версией. Прямой перенос Минутки 1 на `5m`
сломал преимущество: уже на 30 днях результат отрицательный, а на 180 и 365
днях просадка становится критической.

Главная причина: та же механика на более крупной свече дает меньше точных
микро-входов, хуже win rate и более тяжелые time-stop/stop-loss выходы.

Практический вывод: для `5m` нельзя просто копировать `1m`-логику. Нужны
отдельные параметры под пятиминутный таймфрейм: другой фильтр сигнала, другой
тейк/стоп, меньший размер позиции и, возможно, режимный фильтр.
