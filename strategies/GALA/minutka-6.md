# Минутка 6

## Название

GALAUSDT 1m combined S3 + G5

## Идея

Минутка 6 объединяет две high win rate минутки:

- SHORT-часть из минутки 3.
- LONG-часть из минутки G5.

Цель объединения: получить более сбалансированную минутку, которая может
торговать и LONG, и SHORT, но держит только одну позицию одновременно.

## Общие параметры

| Параметр | Значение |
|---|---:|
| Пара | `GALAUSDT` |
| Таймфрейм | `1m` |
| Направление | `both` |
| Вход | Лимитный у текущей цены |
| Размер позиции | `25%` текущего баланса |
| Комиссия | `0.02%` вход + `0.02%` выход |
| Проскальзывание | `0` |
| Одновременных позиций | `1` |

## LONG-часть

| Параметр | Значение |
|---|---:|
| Источник | Минутка G5 |
| Фильтр сигнала | `long_score >= 50` |
| Тейк | `0.25%` |
| Стоп | `4%` |
| Выход по времени | `240 минут` |

## SHORT-часть

| Параметр | Значение |
|---|---:|
| Источник | Минутка 3 |
| Фильтр сигнала | `short_score >= 40` |
| Тейк | `0.25%` |
| Стоп | `3%` |
| Выход по времени | `180 минут` |

## Команда

```bash
python3 scripts/gala_mb_backtest.py --market futures_archive --days 365 --direction both \
  --entry-mode maker_limit \
  --limit-entry-offset-pct 0 \
  --limit-entry-timeout-min 1 \
  --fee-pct 0.0002 \
  --slippage-pct 0 \
  --position-pct 0.25 \
  --long-threshold 50 \
  --long-tp-pct 0.0025 \
  --long-sl-pct 0.040 \
  --long-time-stop-min 240 \
  --short-threshold 40 \
  --short-tp-pct 0.0025 \
  --short-sl-pct 0.030 \
  --short-time-stop-min 180
```

## Вывод

Минутка 6 хорошо выглядит на 90 днях, но не проходит проверку на 180 и 365
днях. Простое объединение LONG и SHORT не решает проблему плохих рыночных
режимов: обе части минутки могут одновременно терять edge.

Минутка сохранена как исследовательский вариант. Для превращения ее в рабочую
нужен фильтр рыночного режима, который будет отключать торговлю в периоды, где
микроскальп теряет математическое ожидание.
